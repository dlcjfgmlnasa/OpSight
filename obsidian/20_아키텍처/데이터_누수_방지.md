# 데이터 누수 방지 (data leakage)

> 시뮬레이션 시간 `t` 에서 `t` 이후 데이터 절대 X. brief §13.2.

## 데이터 누수란

ML / agent 평가에서 *시간 흐름을 어긴 정보* 사용.

예: "10:50 시점 hypotension risk 예측" 인데 11:05 ABP 값을 봄 → 누수.

결과:
- 모델 성능 *과대 평가*
- 실제 배포 시 미래 정보 못 봐 실패
- 시뮬레이션 신뢰도 파괴

## OpSight invariant

```
모든 tool 은 SimClock.now_s 인지.
query_window_end_s > clock.now_s 면 거부.
```

시스템 어디서도 미래 데이터 읽지 못함.

## 구현 — `SimClock.assert_le`

```python
# opsight/sim_clock.py

class SimClock:
    def assert_le(self, query_window_end_s: float) -> None:
        if query_window_end_s > self._now_s:
            raise ValueError(
                f"data leakage: query_window_end_s={query_window_end_s} "
                f"exceeds current sim_time={self._now_s}"
            )
```

조용한 통과 X. Hard fail.

## Tool wrapper leakage guard

모든 EMR tool (FM tool 도) `_leakage_guard` 호출. 위반 시 `ToolError` 반환:

```python
# opsight/tools/emr_tools_stub.py

def _leakage_guard(request, clock, query_window_end_s):
    if query_window_end_s > clock.now_s:
        return ToolResponse(
            error=ToolError(
                type="leakage_violation",
                message=f"query_window_end_s={query_window_end_s} exceeds clock.now_s={clock.now_s}",
            ),
            ...
        )
    return None
```

## sim_time vs wall_time

```
wall_time:  실제 사람 시간 (11:30:00.123)
sim_time :  시뮬레이션 안 환자 시간 (case 시작 후 90.5s)
```

같은 wall_time 에 여러 case 동시 시뮬레이션 가능. 각 case 는 *자기 시간* 을 가진다.

```python
clock = SimClock(start_s=0.0)
clock.tick(30.0)   # sim_time → 30.0
clock.tick(30.0)   # sim_time → 60.0
```

매 tick 30s sim 진행. Wall_time 은 (mock 일 때) ms 단위. 진짜 LLM 합류 시 sim 과 비슷한 수준 가능.

## Wall 이 sim 추월 시 — 실시간성 손실

30s sim tick 처리에 wall 35s 면 점점 뒤쳐짐.

```python
@dataclass
class TickMeasurement:
    sim_time_before_s: float
    sim_time_after_s: float
    wall_start: float
    wall_end: float

    @property
    def wall_elapsed_s(self) -> float:
        return self.wall_end - self.wall_start
```

`TickMeasurement.wall_elapsed_s` 측정 → shallow 15s budget 검증 (brief §6.1). 현재 mock + placeholder: ~5ms. Real Light LLM: ~3–8s 예상 — budget 안.

## "실시간" 정의

- **Simulated real-time** — 과거 VitalDB recording 을 시간 순서대로 stream. `t` 시점에 `t` 이전만 보임.
- **Prospective** (실제 환자에 붙어 실시간) — PoC 범위 X. NRF 도전형 2–3년차 후속 과제.

Brief §10.

## 어떤 패턴이 누수인가

| 패턴 | 누수 |
|------|------|
| Tool 9 `time_window.end > clock.now_s` | ✅ 발생 → `leakage_violation` |
| FM 입력 신호가 미래 sample 포함 | ✅ 발생 (signal ingest 단계에서 차단 의무) |
| `forecast_signal` 이 미래 *예측* 반환 | ❌ OK — 미래 *예측* 은 정상 |
| State 의 `risk_history` 에 *현재 tick 추가 sample* | ❌ OK — 현재 tick 까지 모두 OK |

→ **금지 = 미래의 실측 데이터.** 미래의 예측은 허용.

## Signal ingest 단계에서도 차단

Tool layer 의 leakage 만 강제. **신호 ingest 자체** (signal-ingest-engineer 영역) 도 30s window 안 sample 만 가져와야 함. Ingest 가 미래 sample 끼우면 tool layer 못 막음.
