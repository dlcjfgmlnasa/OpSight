# 00. OpSight 프로젝트 개요

## 한 줄 정체성

**OpSight** — 수술 중 환자의 생체 신호를 보고 "지금 / 곧 위험한가?" 를 임상의에게 알려주는 **tool-using LLM agent**.

영문 정식 표기:
> *A universal, modality-agnostic, quality-aware LLM agent for real-time intraoperative hemodynamic reasoning, powered by a cross-domain pretrained multimodal biosignal foundation model.*

핵심 키워드:
- **Universal** — 비심장 (non-cardiac) 주요 수술 전반에 동작
- **Modality-agnostic** — 가용한 신호 (ABP / ECG / PPG / EEG 등) 만으로 작동
- **Quality-aware** — 신호 품질이 낮을 때 불확실성을 정직하게 표현
- **Surgery-aware** — 수술 유형 / phase 에 따라 추론 적응
- **Real-time (simulated)** — 30초 cycle + on-demand deep brief

## 어떻게 작동하는가

```
[VitalDB 생체 신호] ──► [Foundation Model (FM)] ──► [21개 tool 결과]
                                                            │
                                                            ▼
                                                      [LLM 추론]
                                                            │
                          ┌─────────────────────────────────┴─┐
                          ▼                                   ▼
                   [Shallow narration]                  [Deep brief]
                   (30s, 1문장 한글)                  (event, 9-section)
```

자세한 구조: [[20_아키텍처/Dual_mode_architecture]].

## 왜 만드는가

수술 중 저혈압 (hypotension) 등 혈역학 (hemodynamic) 이벤트는 술후 합병증의 큰 원인. 기존 monitor / early-warning 의 한계:

- 단일 modality (ABP only 의 Hatib 등)
- 신호 품질 무시 (NaN 도 risk 0.5 그대로 출력)
- 수술 유형 무시
- 시간 압박 하 임상의에게 해석 어려움

OpSight 는 이 4 가지를 모두 해결. 자세한 motivation: `docs/project_brief.md §2`.

## 데이터셋 / 평가

- **VitalDB** — SNUH 의 6,388 case open dataset (비심장 수술). https://vitaldb.net
- 코호트 정책 적용 후 약 **5,946 case** (수술 < 30분 제외)
- 평가: 자동 metric + LLM-as-judge + **이형철 교수님 그룹 anesthesiologist 5–7명 평가** (200–300 brief)
- 목표 venue: **npj Digital Medicine**

## 핵심 정책 (반드시 알아야 할 것)

1. **임상 사실 가드** — agent 는 임상 진단을 단정하지 않음. 모든 단정은 `[CLINICIAN-REVIEW: 이형철 교수님 그룹 검토 필요]` marker.
2. **데이터 누수 금지** — 시뮬레이션 시간 `t` 에서 `t` 이후 데이터 절대 X.
3. **Trigger 는 rule-based** — Shallow → Deep 분기는 LLM 이 결정하지 않음. 7 rule + 60s cooldown.
4. **한글 우선 보고** — 사용자 / 임상의 출력은 기본 한글. 코드 / paper 는 영문 OK.
