# 00. VitalAgent 프로젝트 개요

## 한 줄 정체성

**VitalAgent**는 수술 중 환자의 생체 신호를 보고 "지금 위험한가? 곧 위험해질 것 같은가?"를 임상의 (clinician)에게 알려주는 **tool-using LLM agent**다.

영문 정식 표기:
> *A universal, modality-agnostic, quality-aware LLM agent for real-time intraoperative hemodynamic reasoning, powered by a cross-domain pretrained multimodal biosignal foundation model.*

핵심 키워드:
- **Universal** — 비심장 (non-cardiac) 주요 수술 전반에 동작
- **Modality-agnostic** — 가용한 신호 (ABP / ECG / PPG / EEG 등)만으로 작동
- **Quality-aware** — 신호 품질이 낮을 때 불확실성을 정직하게 표현
- **Surgery-aware** — 수술 유형 / phase에 따라 추론 적응
- **Real-time (simulated)** — 30초 cycle + on-demand deep brief

## 어떻게 작동하는가 (간단히)

```
[VitalDB 생체 신호] ──► [Foundation Model (FM)] ──► [21개 tool 결과]
                                                            │
                                                            ▼
                                                      [LLM 추론]
                                                            │
                                                            ▼
                          ┌────────────────────────────────┴───┐
                          ▼                                    ▼
                   [Shallow narration]                   [Deep brief]
                   (30초마다, 1문장 한글)              (event 발생 시, 9-section)
```

자세한 구조는 [[20_아키텍처/Dual_mode_architecture]] 참조.

## 왜 만드는가 (Why)

수술 중 저혈압 (hypotension) 등의 혈역학적 (hemodynamic) 이벤트는 술후 합병증의 큰 원인인데, 기존 monitor / early-warning 시스템은:

- 단일 modality (ABP만 보는 Hatib 등)
- 신호 품질에 침묵 (NaN이어도 risk 0.5 같은 값 그대로 출력)
- 수술 유형 무시
- 시간 압박 하의 임상의에게 해석 어려움

VitalAgent는 이 4가지를 모두 해결하려는 시도. 자세한 motivation은 `docs/project_brief.md §2` 참조.

## 데이터셋 / 평가

- **VitalDB** — SNUH의 6,388 case open dataset (비심장 수술). https://vitaldb.net
- 코호트 정책 적용 후 약 **5,946 case** (수술 < 30분 제외)
- 평가: 자동 metric + LLM-as-judge + **이형철 교수님 그룹의 5–7명 anesthesiologist 평가** (200–300 brief)
- 목표 venue: **npj Digital Medicine**

## 현재 진행 상황

자세한 내용은 [[40_플랜/진행상황]]에 풀어 썼다. 한 문단으로 요약하면:

진짜 Foundation Model이 아직 학습 중이라 자리를 **Mock FM (Tier 1 stub / Tier 2 rule-based / Tier 3 Light ML)** 으로 채워두었고, 그 자리 위에서 dual-mode (shallow 30초 + deep on-demand) graph가 100 case 끝까지 돈다. **21개의 tool** 이 모두 코드로 존재하고 (일부 stub), system prompt 한글 7종 + 9-section brief generator도 작성되어 있다. **누적 테스트 217개가 통과**한다. 다음 큰 작업은 placeholder LLM 자리에 vLLM Llama를 끼우는 것 + 이형철 교수님 그룹 회의 결과를 받아 코호트 / surgery context / intervention response head 설계를 확정하는 것.

## 핵심 정책 한 줄씩 (반드시 알아야 할 것)

1. **임상 사실 가드** — Agent는 임상 진단을 단정하지 않는다. 모든 단정은 `[CLINICIAN-REVIEW: 이형철 교수님 그룹 검토 필요]` marker로 처리.
2. **데이터 누수 금지** — 시뮬레이션 시간 `t`에서 `t` 이후 데이터를 절대 읽지 않는다.
3. **Trigger는 rule-based** — Shallow → Deep 분기는 LLM이 결정하지 않는다. 7개 규칙 + 60초 cooldown.
4. **한글 우선 보고** — 사용자 / 임상의 출력은 기본 한글. 코드 / paper는 영문 OK.
